# 쿠키

- 사용자 브라우저에 저장되는 로컬 데이터 파일

# 세션

- 사용자의 첫 요청을 받으면 서블릿단에서 먼저 세션을 생성하여, 해당 내용을 ServletContext에 인스턴스를 생성한다
- 생성한 인스턴스는 WAS의 메모리에 올라간다
    - 그래서 세션의 최대 생명주기는 WAS의 동작기간까지이다
- 그렇게 만들어진 세션에는 고유 세션 id가 존재한다
- 첫 요청에 대한 응답으로 response header의 set-cookie란에 session id를 응답하여 세션 id를 쿠키로 생성하여 지니게 한다
- 다음 모든 요청부터는 request header의 cookie란에 해당 세션 id 쿠키가 요청에 매번 같이 포함되게 된다
- 그 후로 서블릿에서 해당 쿠키의 세션 id를 통해 해당 사용자에 대한 세션을 같이 응답해주므로서, 세션이 각각의 요청마다 계속 따라다닐수 있게 된다

# 세션 값 접근

- JSP, 톰캣 환경에서는 톰캣이 WAS 역할을 수행하기 때문에 세션이 톰캣 메모리에 올라간다
- JPS에서 페이지를 렌더링할때는 톰캣이 관여한다
- 페이지 렌더링을 톰캣이 수행하는데 세션 메모리도 톰캣에 있으니까 세션값도 문제없이 접근이 가능하나보다... 라고 이해했었다
